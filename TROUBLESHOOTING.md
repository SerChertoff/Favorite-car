# Решение проблем с белым экраном на GitHub Pages

## Диагностика

1. **Откройте консоль браузера (F12)** и проверьте ошибки:
   - Если видите ошибки 404 для JS/CSS файлов - проблема с base path
   - Если видите ошибки JavaScript - проблема в коде
   - Если нет ошибок, но белый экран - проблема с монтированием Vue

2. **Проверьте URL:**
   - Должен быть: `https://username.github.io/Favorite-car/`
   - НЕ должен быть: `https://username.github.io/` (без имени репозитория)

## Решения

### 1. Убедитесь, что base path правильный

Проверьте в `vite.config.ts`:
```typescript
const base = process.env.VITE_BASE_PATH || 
  (process.env.NODE_ENV === "production" ? "/Favorite-car/" : "/");
```

**Важно:** Имя должно точно совпадать с именем репозитория (с учетом регистра!)

### 2. Проверьте настройки GitHub Pages

1. Settings → Pages
2. Source: выберите "GitHub Actions" или "Deploy from a branch"
3. Если используете ветку: выберите `gh-pages` и папку `/ (root)`

### 3. Очистите кэш браузера

- Ctrl+Shift+R (Windows/Linux)
- Cmd+Shift+R (Mac)

### 4. Проверьте логи сборки

В репозитории: Actions → последний workflow → проверьте логи на ошибки

### 5. Локальная проверка

```bash
npm run build
npm run preview
```

Откройте `http://localhost:4173/Favorite-car/` и проверьте, работает ли

## Частые проблемы

### Проблема: Ошибки 404 для assets

**Решение:** Base path не совпадает с именем репозитория. Проверьте регистр букв!

### Проблема: Белый экран без ошибок

**Решение:** 
1. Проверьте консоль браузера на наличие ошибок
2. Убедитесь, что элемент `#app` существует в HTML
3. Проверьте, что Vue приложение монтируется

### Проблема: Работает локально, но не на GitHub Pages

**Решение:**
1. Убедитесь, что `NODE_ENV=production` при сборке
2. Проверьте, что base path установлен правильно
3. Убедитесь, что все файлы закоммичены
